# Розгортання та процес розробки (Developer Setup & Flow)

Цей документ описує повний процес для розробників: від завантаження репозиторію до створення Pull Request (PR) у проєкт Excalidraw.

## 1. Підготовка (Prerequisites)

Перед початком переконайтеся, що у вас встановлені необхідні інструменти:

- **Node.js**: мінімальна версія `>=18.0.0`.
- **Yarn**: проєкт використовує Yarn версії `1.22.x` як пакетний менеджер (це monorepo).
- **Git**: для контролю версій.

## 2. Завантаження коду (Download & Clone)

Оскільки ви плануєте вносити зміни та створювати PR, почніть зі створення форку оригінального репозиторію.

1. Перейдіть на GitHub та зробіть **Fork** репозиторію Excalidraw у свій акаунт.
2. Склонуйте ваш форк локально на комп'ютер:

   ```bash
   git clone https://github.com/<ваш-username>/excalidraw.git
   cd excalidraw
   ```

3. Налаштуйте оригінальний репозиторій як `upstream`, щоб мати змогу синхронізуватись із майбутніми оновленнями:

   ```bash
   git remote add upstream https://github.com/excalidraw/excalidraw.git
   git fetch upstream
   ```

## 3. Встановлення залежностей (Install)

Щоб встановити всі необхідні пакети для всіх частин monorepo (`excalidraw-app`, `packages/*`, `examples/*`), виконайте команду:

```bash
yarn install
```

## 4. Запуск проєкту (Run)

Для локального запуску вебдодатка в режимі розробки (Development mode) використовуйте:

```bash
yarn start
```

Після виконання команди додаток запуститься (зазвичай за адресою `http://localhost:3000` або `http://localhost:5173` залежно від Vite конфігурації) і сторінка автоматично відкриється у браузері.

## 5. Внесення змін (Make a Change)

1. **Створення гілки:** Завжди створюйте нову гілку для своїх змін від найсвіжішої версії `master`.

   **Правила іменування гілок (Branch Naming Conventions):** Рекомендується використовувати префікси, що відповідають типу змін (засновано на Conventional Commits/Branches):

   - `feat/` або `feature/` — для розробки нового функціоналу (наприклад, `feat/add-new-tool`).
   - `fix/` або `bugfix/` — для виправлення помилок (наприклад, `fix/canvas-crash`).
   - `docs/` — для написання або оновлення документації.
   - `refactor/` — для покращення структури існуючого коду без зміни його поведінки.
   - `chore/` — для рутинних задач, зміни конфігурації або оновлення залежностей.

   ```bash
   git checkout master
   git pull upstream master
   git checkout -b feat/<коротка-назва-фічі>
   ```

2. **Написання коду:** Внесіть необхідні зміни. Зверніть увагу на структуру:
   - Основний код вебдодатка знаходиться у `excalidraw-app/`.
   - Основна бібліотека та компоненти знаходяться у `packages/excalidraw/`.
3. **Перевірка та форматування:** Перед створенням коміту переконайтеся, що код відповідає стандартам проєкту:

   ```bash
   yarn fix             # Запуск Prettier та ESLint з автоматичним фіксом помилок
   yarn test:typecheck  # Перевірка типів TypeScript
   yarn test:all        # Запуск усіх тестів (Unit тести, лінтери, форматування)
   ```

## 6. Створення Pull Request (Create a PR)

Коли ваші зміни готові, протестовані та перевірені:

1. **Коміт змін:** Додайте файли та зробіть коміт (використовуйте зрозумілі повідомлення):

   ```bash
   git add .
   git commit -m "feat: допис короткого опису вашої зміни"
   ```

2. **Відправка коду (Push):** Завантажте вашу гілку до вашого форку на GitHub:

   ```bash
   git push origin <назва-вашої-гілки>
   ```

3. **Створення PR:**
   - Зайдіть на GitHub сторінку вашого форку або оригінального репозиторію Excalidraw.
   - Ви побачите зелену кнопку-підказку "Compare & pull request". Натисніть її.
   - Детально опишіть ваші зміни у запропонованому шаблоні PR, додайте скриншоти, якщо це стосується UI.
   - Натисніть **"Create pull request"**.

🎉 Готово! Тепер залишається лише дочекатися рев'ю від майнтейнерів проєкту.
