# Product Requirements Document (PRD) — Excalidraw

## 1. Мета продукту (Product Goal)
Створити швидкий, інтуїтивно зрозумілий інструмент (virtual whiteboard) для створення ескізів, діаграм та ілюстрацій, який імітує малювання від руки. Продукт має забезпечувати мінімальне тертя при початку роботи (без обов'язкової реєстрації) та підтримувати як індивідуальну роботу, так і спільну роботу в реальному часі (real-time collaboration), фокусуючись на безпеці та конфіденційності користувачів.

## 2. Цільова аудиторія (Target Audience)
*   **Інженери-програмісти та архітектори:** Для швидкого створення технічних діаграм, архітектури систем, блок-схем.
*   **Дизайнери та продакт-менеджери:** Для створення wireframes, швидких UI/UX прототипів та проведення мозкових штурмів.
*   **Викладачі, ментори та студенти:** Для пояснення складних концепцій за допомогою візуальних матеріалів під час онлайн-уроків чи лекцій.
*   **Віддалені та розподілені команди:** Для проведення спільних сесій планування, ретроспектив, візуалізації ідей та синхронізації під час дзвінків.

## 3. Ключові функції (Key Features)
*   **Малювання базових фігур:** Лінії, стрілки, прямокутники, еліпси, ромби зі стилізацією.
*   **Рукописне введення (Freedraw):** Можливість малювати довільні лінії за допомогою миші, графічного планшета, стилуса або сенсорного екрану.
*   **Робота з текстом:** Додавання тексту з вибором стилю (стилізований під рукописний шрифт, моноширинний, звичайний).
*   **Спільна робота в реальному часі (Collaboration):** Одночасне спільне редагування полотна з відображенням позицій курсорів інших користувачів.
*   **End-to-End Encryption (E2EE):** Наскрізне шифрування під час спільних сесій та збереження малюнків за посиланням — гарантія того, що сервер не має доступу до вмісту діаграм.
*   **Бібліотека компонентів:** Збереження власних наборів елементів або встановлення готових публічних бібліотек (Excalidraw Libraries).
*   **Експорт та інтеграція:** Збереження малюнків у форматах PNG, SVG, буфер обміну, та у форматі `.excalidraw`. Підтримка темної та світлої теми для експортованих зображень.
*   **Локальне збереження:** Автоматичне збереження поточного стану полотна локально в браузері (Local Storage/IndexedDB), відновлення сесії після перезавантаження.
*   **Розумні зв'язки (Smart Bindings):** Прикріплення тексту і стрілок до фігур для зручної побудови та переміщення блок-схем і графів.
*   **Zen Mode та View Mode:** Режим зосередження з прихованим інтерфейсом та режим тільки для читання.

## 4. Технічні обмеження (Technical Constraints)
*   **Продуктивність рендерингу:** Додаток має працювати швидко і плавно (бажано 60 FPS) при великій кількості об'єктів на полотні. Основна візуалізація здійснюється переважно через HTML5 Canvas API (Rough.js для стилізації).
*   **Offline-first / Local-first архітектура:** Основний функціонал повинен працювати без підключення до інтернету (PWA, Service Workers) як повноцінний локальний додаток.
*   **Кросбраузерність:** Повинна підтримуватися стабільна робота в усіх сучасних версіях популярних браузерів (Chrome, Firefox, Safari, Edge) на різних платформах (Desktop, Mobile) без плагінів.
*   **Зберігання стану (State Management):** Ефективне та передбачуване управління складним мутабельним і імутабельним станом сцени, підтримка історії дій (Undo/Redo).
*   **Обсяг локальних даних:** Обмеження на обсяг даних, які можуть бути надійно збережені в локальному сховищі браузера.
*   **Клієнтське шифрування:** Усі криптографічні операції (E2EE) мають виконуватися виключно на стороні клієнта за допомогою стандартних Web Crypto API без витоку ключів на сервер.
*   **Розмір бандла:** Необхідність мінімізації розміру початкового завантаження для забезпечення швидкого рендеру сторінки (First Contentful Paint).
