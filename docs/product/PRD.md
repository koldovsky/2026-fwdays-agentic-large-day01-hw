# Product Requirements Document (PRD) — Excalidraw

## 1. Мета продукту (Product Goal)
Створити швидкий, інтуїтивно зрозумілий інструмент (virtual whiteboard) для створення ескізів, діаграм та ілюстрацій, який імітує малювання від руки. Продукт має забезпечувати мінімальне тертя при початку роботи (без обов'язкової реєстрації) та підтримувати як індивідуальну роботу, так і спільну роботу в реальному часі (real-time collaboration), фокусуючись на безпеці та конфіденційності користувачів за допомогою End-to-End Encryption (E2EE) для спільних сесій.

## 2. Цільова аудиторія (Target Audience)
*   **Інженери-програмісти та архітектори:** Для швидкого створення технічних діаграм, архітектури систем, блок-схем.
*   **Дизайнери та продакт-менеджери:** Для створення wireframes, швидких UI/UX прототипів та проведення мозкових штурмів.
*   **Викладачі, ментори та студенти:** Для пояснення складних концепцій за допомогою візуальних матеріалів під час онлайн-уроків чи лекцій.
*   **Віддалені та розподілені команди:** Для проведення спільних сесій планування, ретроспектив, візуалізації ідей та синхронізації під час дзвінків.
*   **Вільні художники та ентузіасти:** Для швидких начерків та візуалізації поточних ідей без зайвих налаштувань.

## 3. Ключові функції (Key Features)
*   **Малювання базових фігур:** Лінії, стрілки, прямокутники, еліпси, ромби зі стилізацією під намальовані від руки (за допомогою Rough.js).
*   **Рукописне введення (Freedraw):** Можливість малювати довільні лінії за допомогою миші, графічного планшета, стилуса або сенсорного екрану зі згладжуванням кривих.
*   **Робота з текстом:** Додавання вільно розміщеного тексту або тексту всередині фігур з вибором стилю (стилізований під рукописний шрифт, моноширинний, звичайний), кольору, розміру та вирівнювання.
*   **Спільна робота в реальному часі (Collaboration):** Одночасне спільне редагування полотна з відображенням позицій курсорів інших користувачів, обміном повідомленнями через курсор.
*   **End-to-End Encryption (E2EE):** Наскрізне шифрування під час спільних сесій та збереження малюнків за посиланням — гарантія того, що сервер не має доступу до вмісту діаграм, оскільки ключі генеруються і передаються у фрагменті URL.
*   **Бібліотека компонентів:** Збереження власних наборів елементів або встановлення готових публічних бібліотек (Excalidraw Libraries) для підвищення швидкості побудови діаграм.
*   **Експорт та інтеграція:** Збереження малюнків у форматах PNG, SVG, буфер обміну, та у форматі `.excalidraw`. Підтримка темної та світлої теми для експортованих зображень, можливість експортувати лише виділені об'єкти.
*   **Локальне збереження:** Автоматичне збереження поточного стану полотна локально в браузері (Local Storage/IndexedDB), відновлення сесії після перезавантаження сторінки або закриття браузера.
*   **Розумні зв'язки (Smart Bindings):** Прикріплення тексту до фігур та стрілок до центрів чи граней об'єктів для зручної побудови та автоматичного переміщення блок-схем і графів.
*   **Додаткові інструменти:** Гумка (eraser) для видалення елементів, інструмент вибору (selection tool) для групового переміщення або трансформації об'єктів.
*   **Zen Mode та View Mode:** Режим зосередження з прихованим інтерфейсом (без панелей інструментів) та режим тільки для читання (View Mode).

## 4. Цільовий обсяг (Target Scope)
Excalidraw фокусується на швидкому, інтуїтивному малюванні та легкому обміні сесіями. У цільовий обсяг (in-scope) входить:
*   Реалізація малювання 2D-примітивів: фрейми, текст, геометричні фігури, стрілки, малювання від руки.
*   Імітація стилю малювання "від руки" (hand-drawn).
*   Безпечна та швидка синхронізація полотна між кількома користувачами (WebSockets + WebRTC, E2EE).
*   Локальне збереження стану та можливість експорту/імпорту даних полотна.
*   Миттєвий запуск без вимоги створювати акаунт.
*   Підтримка ком'юніті-бібліотек (Excalidraw Libraries) для поширення та зручного використання шаблонів.
*   Підтримка розширень (packages) для інтеграції Excalidraw як React-компонента (`@excalidraw/excalidraw`) у сторонні додатки.

## 5. Що продукт НЕ робить (Non-goals / Out of Scope)
Для підтримки легкості продукту та уникнення "feature creep" (перевантаження функціоналом), Excalidraw свідомо відмовляється від наступного:
*   **Не є заміною Figma або Sketch:** Excalidraw не призначений для "pixel-perfect" дизайну інтерфейсів, векторної ілюстрації високої точності чи верстки.
*   **Не є повноцінною Kanban чи Jira-дошкою:** Не підтримує складного управління проектами, статусами задач чи assignee.
*   **Не є інструментом для 3D:** Платформа працює суто в 2D-просторі, без підтримки 3D-моделювання чи рендерингу.
*   **Не вимагає реєстрації для базового функціоналу:** Основний фокус на тому, щоб користувач міг почати малювати одразу після переходу на сайт. Облікові записи не є обов'язковими (хоча можуть пропонуватись в Excalidraw+, це виходить за рамки відкритого ядра).
*   **Не зберігає відкритий текст та дані діаграм на серверах спільної роботи:** Сервер виступає лише реле для зашифрованих повідомлень. Продукт технічно не може і не буде аналізувати контент малюнків під час спільних сесій.
*   **Не підтримує складні градієнти чи маски:** Стилізація обмежується базовими суцільними кольорами, штрихуванням або патернами Cross-hatch для збереження фірмового вигляду.

## 6. Технічні обмеження (Technical Constraints)
*   **Продуктивність рендерингу:** Додаток має працювати швидко і плавно (бажано 60 FPS) при великій кількості об'єктів на полотні. Основна візуалізація здійснюється переважно через HTML5 Canvas API (Rough.js для стилізації).
*   **Offline-first / Local-first архітектура:** Основний функціонал повинен працювати без підключення до інтернету (PWA, Service Workers) як повноцінний локальний додаток.
*   **Кросбраузерність:** Повинна підтримуватися стабільна робота в усіх сучасних версіях популярних браузерів (Chrome, Firefox, Safari, Edge) на різних платформах (Desktop, Mobile) без плагінів.
*   **Зберігання стану (State Management):** Ефективне та передбачуване управління складним мутабельним і імутабельним станом сцени, підтримка історії дій (Undo/Redo).
*   **Обсяг локальних даних:** Обмеження на обсяг даних, які можуть бути надійно збережені в локальному сховищі браузера.
*   **Клієнтське шифрування:** Усі криптографічні операції (E2EE) мають виконуватися виключно на стороні клієнта за допомогою стандартних Web Crypto API без витоку ключів на сервер.
*   **Розмір бандла:** Необхідність мінімізації розміру початкового завантаження для забезпечення швидкого рендеру сторінки (First Contentful Paint).
*   **Відсутність серверного рендерингу (SSR):** Excalidraw є переважно Client-Side Rendered (CSR) додатком, що впливає на способи ініціалізації Canvas і вимагає уваги до SEO, де це є можливим поза самим полотном.

## 7. Відкритий вихідний код (Open Source / Business Model Constraints)
*   **MIT License:** Ядро продукту має залишатися open-source за ліцензією MIT, забезпечуючи свободу для розробників та спільноти.
*   **Взаємодія зі спільнотою:** Інструмент має залишатися дружнім до контриб'юторів, що передбачає підтримання високого стандарту чистоти коду, типізації (TypeScript) та використання популярних інструментів екосистеми (React).
*   **Бізнес-модель:** Важливо розмежовувати безкоштовний відкритий продукт (`excalidraw.com`) і комерційний сервіс (Excalidraw+). PRD визначає межі відкритої версії, що забезпечує базову цінність малювання та співпраці.
