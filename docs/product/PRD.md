# Product Requirements Document (reverse-engineered)

Документ відновлений із фактичної структури репозиторію `excalidraw-monorepo`, коду `excalidraw-app`, пакетів `@excalidraw/*` та внутрішньої документації (`docs/memory/*`, `docs/technical/architecture.md`). Він описує продукт «як є», а не оригінальний бізнес-замовлення.

---

## Мета продукту

1. **Веб-редактор дошки** — надати в браузері швидке створення схем, діаграм і нотаток «від руки» з візуальною мовою sketch (Rough.js), мінімальним тертям у робочому процесі та підтримкою фокусу (Zen, перегляд без редагування).

2. **Перевикористовуваний React SDK** — експортувати `@excalidraw/excalidraw` як компонент і імперативний API для вбудовування білої дошки в сторонні застосунки без прив’язки до конкретного хостингового бекенду цього репозиторію.

3. **Модульна доменна база** — тримати спільну логіку сцени, математики та утиліт у окремих пакетах (`@excalidraw/common`, `@excalidraw/element`, `@excalidraw/math`, `@excalidraw/utils`), щоб ядро редактора й приклади інтеграції (`examples/*`) будувалися на однакових примітивах.

4. **Сценарій «повного» веб-клієнта** — у шарі `excalidraw-app` додати колаборацію в реальному часі, обмін посиланнями, персистенцію та інтеграції з зовнішніми сервісами (на кшталт WebSocket і Firebase), тоді як сам пакет редактора залишається переносимим.

## Цільова аудиторія

| Сегмент | Потреба, яку закриває продукт |
|--------|-------------------------------|
| **Індивідуальні користувачі та команди** | Швидкі креслення, пояснення ідей, спільне редагування дошки через кімнату/посилання; експорт артефактів для передачі поза сесією. |
| **Продуктові команди та фасилітатори** | Режими перегляду та мінімалістичний UI для презентацій; лазерний вказівник і спільні курсори для живих сесій. |
| **Розробники та інтегратори** | Вбудовування `<Excalidraw />` у власні React-застосунки з налаштуванням UI (`UIOptions`), режимів view/zen, підпискою на зміни (`onChange`, імперативний API). |
| **Власники інфраструктури** | Статичний фронтенд, який можна зібрати (Vite), обслуговувати через Nginx (у т.ч. у Docker) без обов’язкового Node API в цьому репозиторії. |

## Ключові функції

### Редактор і полотно

- Інструменти малювання: фігури, лінії, стрілки, текст, зображення, фрейми, ластик, лазер, «рука»; прив’язки, допоміжні лінії, масштаб і прокрутка.
- Робота з елементами: виділення, групування, вирівнювання, копіювання/вставка, undo/redo через централізований шлях дій (`ActionManager` → `syncActionResult`).
- Доступ до дій: панелі інструментів, контекстне меню, гарячі клавіші, Command Palette, пошук по елементах на сцені.

### Досвід і налаштування

- Теми (світла / темна / системна), локалізація інтерфейсу, адаптивність зокрема для мобільних сценаріїв.
- Бібліотека елементів: збережені набори, імпорт бібліотек (у т.ч. з URL-токенів там, де це реалізовано в додатку).
- Експорт і імпорт: зображення, SVG, JSON сцени; у застосунку — додаткові сценарії збереження та інтеграцій залежно від конфігурації.

### Веб-застосунок `excalidraw-app`

- Жива колаборація: синхронізація сцени, відображення учасників (курсори тощо) через зовнішній WebSocket-транспорт (`socket.io-client`).
- Довготривале зберігання та файли: клієнтські інтеграції з Firebase (Firestore, Storage) за наявності налаштувань.
- Меню гамбургер: сцена, експорт, колаборація, палітра команд, пошук, довідка, преференції, тема, мова, фон полотна.

### SDK для хостів

- Публічний компонент `Excalidraw`, провайдери API, реекспорт утиліт відновлення сцени, експорту, серіалізації — для сценарію «whiteboard у своєму продукті».

### Інженерні артефакти

- Приклади інтеграції (`examples/*`, зокрема Next.js).
- Збірка пакетів ESM, типізація TypeScript, тести (Vitest), лінтинг.

## Технічні обмеження

### Архітектура та репозиторій

- **Немає власного backend API в репозиторії** як обов’язкового компонента: колаборація та хмарні сценарії покладаються на зовнішні сервіси та клієнтські інтеграції (`socket.io-client`, Firebase).

- **Розділення бібліотека ↔ хост**: логіка кімнат, шифрування payload перед emit, Firebase — у `excalidraw-app`; ядро редактора не вимагає цих залежностей для базового вбудовування.

- **Залежність `Store` від класу `App`**: на рівні пакетів існує зв’язок доменного сховища з шаром UI редактора, що накладає обмеження на незалежне перевикористання `Store` без контексту редактора.

### Клієнт і середовище виконання

- **Браузерний клієнт**: рендер через HTML5 canvas (подвійний шар — статичний і інтерактивний), Rough.js; цільові браузери зафіксовані в `browserslist` у `excalidraw-app` (без підтримки застарілих IE тощо).

- **Node.js**: `>=18` (кореневий `package.json` та `excalidraw-app`); менеджер пакетів — Yarn 1 (workspaces).

- **Збірка застосунку**: Vite; продакшн — статичні артефакти в `excalidraw-app/build`, типовий деплой — статика + Nginx (Docker-образ копіює збірку в `nginx:alpine`).

### Операційні та мережеві залежності

- Колаборація та спільний доступ **залежать від доступності та політики зовнішніх endpoint’ів** (WebSocket-сервер, Firebase), а не лише від локального статичного хостингу.

- **Sentry та трекінг** у збірці застосунку керуються змінними оточення (наприклад окремий шлях `build:app:docker` з відключенням Sentry).

### Розробка

- У дев-режимі `excalidraw-app` резолвить `@excalidraw/*` на вихідні TypeScript у `packages/` через Vite-аліаси; для публікації пакетів потрібна послідовна збірка `build:common` → … → `build:excalidraw`.
