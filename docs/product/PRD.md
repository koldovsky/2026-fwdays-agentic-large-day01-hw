# Excalidraw — Reverse-Engineered Product Requirements Document

## 1. Призначення продукту

Excalidraw — це open-source віртуальна дошка для створення діаграм та ескізів у характерному стилі «від руки», що працює повністю у браузері. Продукт вирішує задачу швидкої візуальної комунікації з низьким порогом входу — дозволяє накидати ідеї, архітектурні схеми, флоучарти та вайрфрейми без надлишкової складності професійних дизайн-інструментів. Приватність закладена в основу: спільні сесії використовують наскрізне шифрування (AES-GCM), тож навіть сервер не може прочитати вміст малюнків.

## 2. Цільова аудиторія

### Кінцеві користувачі
- Дизайнери, продакт-менеджери та інженери, яким потрібно швидко візуалізувати ідеї під час брейнстормів або зустрічей.
- Викладачі та студенти, що створюють пояснювальні діаграми.
- Будь-хто, кому зручніше працювати на дошці, а не у формальному інструменті для діаграм.

### Розробники-інтегратори
- Команди, що вбудовують React-компонент `@excalidraw/excalidraw` у власні застосунки (платформи документації, нотатники, внутрішні інструменти).
- Пакет надає 40+ пропсів, імперативний API, функції експорту та підкомпоненти (`<Sidebar>`, `<MainMenu>`, `<WelcomeScreen>` тощо) для глибокої кастомізації.

### Команди для спільної роботи
- Розподілені команди, що використовують real-time колаборацію для спільного редагування канвасу з живими курсорами, визначенням неактивності та лазерною вказівкою.

## 3. Ключові можливості

### 3.1 Візуальний стиль «від руки»
Фірмовий вигляд забезпечують **roughjs 4.6.4** (рендеринг на canvas зі «скетчевою» естетикою) та **perfect-freehand 1.2.0** (малювання від руки з чутливістю до натиску). Рендеринг-пайплайни знаходяться у `packages/excalidraw/renderer/` з окремими шляхами для інтерактивного канвасу (`InteractiveCanvas.tsx`), статичного канвасу (`StaticCanvas.tsx`) та SVG-експорту (`staticSvgScene.ts`).

### 3.2 Розвинена система елементів
Визначена у `packages/element/src/types.ts`, редактор підтримує:

- **Базові фігури**: прямокутник, еліпс, ромб, лінія
- **Текст**: багаторядковий, з автозміною розміру, прив'язаний до контейнерів (текст всередині фігур)
- **Стрілки**: прямі та з вигинами (elbowed), 14 типів наконечників, включно з маркерами кардинальності (`one`, `many`, `zero_or_one` тощо) та прив'язкою до елементів (стрілки «чіпляються» до країв фігур)
- **Малювання від руки**: з чутливістю до натиску
- **Зображення**: з кадруванням та контролем пропорцій
- **Фрейми**: організаційні контейнери для групування контенту
- **Вбудовувані елементи / Iframe**: вставка зовнішнього контенту прямо на канвас

Додаткові можливості: групування, блокування, управління z-index (через fractional indexing для безпеки в мультиплеєрі), гіперпосилання на елементи, ласо-виділення та гумка.

### 3.3 Колаборація в реальному часі
Реалізовано у `excalidraw-app/collab/Collab.tsx` та `excalidraw-app/collab/Portal.tsx`:

- **Транспорт**: WebSocket-клієнт Socket.io (`socket.io-client 4.7.2`)
- **Персистентність**: Firebase Firestore для зберігання сцен, Firebase Storage для бінарних файлів (зображень)
- **Вирішення конфліктів**: функція `reconcileElements()` зливає віддалені та локальні зміни
- **Присутність**: живі курсори, визначення неактивності користувача (`IDLE_THRESHOLD`, `ACTIVE_THRESHOLD`), лазерні сліди (`laser-trails.ts`)
- **Безпека**: всі дані шифруються AES-GCM перед відправкою (`packages/excalidraw/data/encryption.ts`)

### 3.4 Вбудовуваний React-компонент (`@excalidraw/excalidraw`)
Опублікований як npm-пакет (v0.18.0), компонент `<Excalidraw />` інтегрується в будь-який React-застосунок:

- **Пропси**: колбеки подій (`onChange`, `onPointerUpdate`, `onPaste`), дані (`initialData`), перемикачі UI (`viewModeEnabled`, `zenModeEnabled`, `gridModeEnabled`, `theme`), слоти для кастомного рендерингу (`renderTopLeftUI`, `renderTopRightUI`, `renderEmbeddable`)
- **Імперативний API**: `updateScene()`, `getSceneElements()`, `setActiveTool()`, `scrollToContent()`, `addFiles()`, керування історією, підписки на події
- **Функції експорту**: `exportToCanvas()`, `exportToBlob()`, `exportToSvg()`, `exportToClipboard()`
- **Обмеження SSR**: завантажується лише на клієнті (`dynamic(() => import(...), { ssr: false })` у Next.js)
- **Приклади**: інтеграція через script-тег (`examples/with-script-in-browser/`), інтеграція з Next.js (`examples/with-nextjs/`)

### 3.5 Можливості експорту
Розташовані у `packages/excalidraw/scene/export.ts` та `packages/excalidraw/clipboard.ts`:

- **SVG**: повний векторний експорт через `renderSceneToSvg()`
- **PNG / JPEG**: растровий експорт через canvas
- **JSON**: нативний формат `.excalidraw` через `serializeAsJSON()`
- **Буфер обміну**: копіювання елементів як PNG або Excalidraw JSON; вставка зображень, тексту та даних Excalidraw

### 3.6 Бібліотеки елементів
Керуються у `packages/excalidraw/data/library.ts` (31 КБ) з UI у `components/LibraryMenu*.tsx`:

- Перегляд, імпорт та публікація колекцій повторно використовуваних елементів
- Інтерфейс `LibraryPersistenceAdapter` для кастомних бекендів зберігання
- Завантаження опублікованих бібліотек за URL
- SVG-превʼю через хук `useLibraryItemSvg`

### 3.7 Офлайн-підтримка та PWA
- **Service Worker**: `vite-plugin-pwa` зі стратегіями кешування для шрифтів, локалей та чанків коду
- **Локальна персистентність**: `EditorLocalStorage.ts` обгортає localStorage; IndexedDB використовується для даних малюнків
- **Встановлюваний додаток**: PWA-маніфест з maskable-іконками (`maskable_icon_x192.png`, `maskable_icon_x512.png`)

### 3.8 Інтернаціоналізація (i18n)
Реалізовано у `packages/excalidraw/i18n.ts` з файлами локалей у `packages/excalidraw/locales/`:

- **46 мов**, включно з RTL-мовами (арабська, іврит, перська, курдська)
- Динамічне перемикання мови через `setLanguage()`
- Хук `useI18n()` та функція `t()` для перекладів
- Поріг включення локалі — 85% завершеності

### 3.9 Text-to-Diagram та AI-можливості
Розташовані у `packages/excalidraw/components/TTDDialog/`:

- **Text-to-Diagram**: природна мова → діаграма через AI-бекенд (ендпоінт конфігурується)
- **Імпорт Mermaid**: автоматичне розпізнавання та конвертація 18+ типів Mermaid-діаграм (flowchart, sequence, class, ER, gantt, mindmap, timeline тощо) — `packages/excalidraw/mermaid.ts`
- **Чат-інтерфейс**: ітеративне вдосконалення діаграм через `ChatInterface.tsx`
- **Керованість**: пропс `aiEnabled` для ввімкнення/вимкнення

### 3.10 Графіки
У `packages/excalidraw/charts/`:

- Стовпчикові діаграми (`charts.bar.ts`)
- Лінійні графіки (`charts.line.ts`)
- Радарні діаграми (`charts.radar.ts`)
- Дані, вставлені з електронних таблиць, автоматично візуалізуються як елементи малюнка

### 3.11 Теми та режими UI
- **Світла / темна тема**: пропс `theme`
- **Zen-режим**: мінімальний UI для малювання без відволікань
- **Режим сітки**: накладення сітки з прив'язкою
- **Прив'язка обʼєктів**: напрямні та лінії прив'язки (`snapping.ts`, 36 КБ)
- **Режим перегляду**: канвас тільки для читання
- **Командна палітра**: пошук та виконання дій з клавіатури

### 3.12 Наскрізне шифрування (E2E)
У `packages/excalidraw/data/encryption.ts`:

- Симетричне шифрування AES-GCM через Web Crypto API
- Генерація 128-бітного ключа, 12-байтний IV
- Усі дані колаборації шифруються до виходу з браузера
- Ключі передаються через фрагмент URL (ніколи не надсилаються на сервер)

## 4. Не-цілі та обмеження

### Чим Excalidraw НЕ є

| Не-ціль | Пояснення |
|---------|-----------|
| **Не інструмент точного дизайну** | Немає редагування векторних шляхів, піксель-перфект вирівнювання чи дизайн-токенів — це свідомо lo-fi. Не заміна Figma, Sketch або Illustrator. |
| **Не 3D-інструмент** | Уся геометрія виключно 2D (`packages/math/src/` покриває точки, лінії, полігони, еліпси — жодних 3D-примітивів). |
| **Не інструмент презентацій** | Немає слайдів, переходів чи режиму доповідача. Лазерна вказівка — це засіб колаборації, а не функція слайдшоу. |
| **Не бекенд / платформа** | npm-пакет не містить серверних компонентів. Персистентність, автентифікація та управління користувачами — відповідальність хост-застосунку. Firebase-інтеграція існує лише у референсному додатку (`excalidraw-app/`). |
| **Не мобільний нативний додаток** | Працює лише у браузері; немає нативних збірок для iOS/Android. Адаптивний UI підлаштовується під мобільні вʼюпорти, але залежить від браузера. |
| **Не SSR-сумісний** | Компонент `<Excalidraw />` потребує браузерний DOM та Canvas API. Серверний рендеринг явно не підтримується. |
| **Без управління документами** | Один канвас на інстанс редактора. Вкладки, сторінки чи організація на рівні проєкту — поза скоупом. |
| **Без контролю доступу** | Колаборація використовує модель спільного ключа кімнати. Ролей, дозволів чи обмежень доступу на рівні користувача немає. |
| **Без offline-first синхронізації** | Хоча localStorage забезпечує базову офлайн-персистентність, немає CRDT-синхронізації. Колаборація потребує активного WebSocket-зʼєднання. |

### Технічні обмеження
- **Підтримка браузерів**: Chrome 70+, Firefox, Safari 12+, Edge 79+ (без IE11)
- **Бюджет бандлу**: 340 КБ (ядро), 290 КБ (локалі), 900 КБ (vendor) — контролюється через `.size-limit.json`
- **Node.js**: >=18.0.0 для розробки; рендеринг у Node.js runtime не підтримується
- **Батьківський контейнер**: повинен мати явну ненульову висоту для відображення канвасу
