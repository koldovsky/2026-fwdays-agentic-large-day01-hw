# Product Context

## Навіщо існує Excalidraw

Інструмент для швидкого малювання схем у стилі "від руки" прямо в браузері.
Без інсталяції, без акаунту — відкрив і малюєш.

Ключова ідея: **hand-drawn стиль знижує бар'єр** — схема виглядає як чернетка,
а не як фінальний дизайн, тому люди менш бояться помилитися або переробити.

## Які проблеми вирішує

| Проблема | Рішення |
|---|---|
| Складні інструменти (Visio, Lucidchart) потребують навчання | Zero-friction UI, мінімум кнопок |
| Діаграми виглядають "занадто офіційно" для чернеток | Hand-drawn стиль через RoughJS |
| Немає спільної роботи в реальному часі | WebSocket collab + Firebase |
| Не можна вбудувати в свій продукт | npm-пакет `@excalidraw/excalidraw` |
| Потрібен офлайн-доступ | PWA з Service Worker |

## Два продукти в одному репо

**excalidraw.com** (excalidraw-app) — кінцевий застосунок для звичайних користувачів.
Має колаборацію, Firebase, Sentry, AI-функції, share-посилання.

**@excalidraw/excalidraw** (npm-пакет) — React-компонент для розробників.
Вбудовується в будь-який React-застосунок. Колаборація і Firebase — не входять,
реалізуються хост-застосунком через `ExcalidrawImperativeAPI`.

## Цільова аудиторія

- **Кінцеві користувачі** — малюють схеми на excalidraw.com
- **Розробники** — вбудовують компонент у власні продукти (Notion, Linear, тощо)
- **Open-source контрибʼютори** — MIT ліцензія, активна спільнота

## UX-принципи

- Нічого не блокує canvas — всі діалоги та меню поверх, не замість
- Infinite canvas — нема меж, нема сторінок
- Все зберігається локально за замовчуванням — приватність з коробки
- Keyboard-first — всі дії доступні з клавіатури
