# Product context — Excalidraw (форк monorepo)

Контекст продукту для UX-рішень і сценаріїв використання. Доповнює `projectbrief.md` саме з точки зору користувача та інтеракцій.

## UX-цілі

- **Швидке малювання** — мінімум кроків від відкриття до першого штриха; інструменти та стилі доступні з панелі та контекстного меню.
- **Відчуття «ручного» скетчу** — roughness, вільні лінії, зрозумілі маніпуляції з елементами на canvas.
- **Керованість сцени** — zoom/pan, сітка, snap, виділення та групування без втрати орієнтації на великих дошках.
- **Експорт і обмін** — PNG/SVG, JSON сцени, можливість вбудовувати сцену в експорт; темна тема для експорту.
- **Спільна робота** — відображення колабораторів, синхронізація сцени (через інтеграції додатку).
- **Фокус** — zen mode, view mode для презентацій без зайвого UI.
- **Доступність вбудовування** — компонент `@excalidraw/excalidraw` має коректно працювати в контейнері з ненульовою висотою та з клієнт-only рендером у SSR-фреймворках.

## Ключові сценарії

1. **Швидкий діаграмний скетч** — користувач обирає інструмент (прямокутник, стрілка, лінія), малює, підключає стрілки до елементів (binding), додає підписи.
2. **Редагування тексту** — подвійний клік / режим тексту, зміна шрифту, вирівнювання, колір з палітри поточних стилів.
3. **Робота з бібліотекою** — додавання готових фігур, повторне використання блоків на дошці.
4. **Фрейми** — групування контенту в frame, керування відображенням (outline, clip, назва).
5. **Експорт** — діалог image/json export, масштаб, фон, опційно embed scene.
6. **Колаборація** — перегляд курсорів/стану інших учасників, узгоджена сцена елементів.
7. **Інтеграція в продукт** — хост-додаток рендерить `<Excalidraw />`, передає початкові елементи та підписується на зміни через API пакета.

## Обмеження UX (що варто враховувати)

- Canvas-орієнтований UI: багато логіки зав’язано на координати сцени, zoom і scroll (`AppState`), а не лише на DOM-tree.
- Складні жести (multi-point лінії, resize, rotate) мають залишатися передбачуваними; стан `newElement` / `multiElement` / `selectionElement` визначає режим інтерпретації вводу.
