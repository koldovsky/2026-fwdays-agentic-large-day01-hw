# Product context (UX/UI та сценарії)

Контекст для цього репозиторію: **Excalidraw** — веб-дошка для швидких схем і малюнків «від руки», плюс **React SDK** (`@excalidraw/excalidraw`) для вбудовування в інші продукти. Нижче — цілі інтерфейсу та типові сценарії користувача, узгоджені з `docs/memory/projectbrief.md`, `docs/technical/architecture.md` та шаром `excalidraw-app`.

---

## Які UX/UI цілі

1. **Швидке малювання без «дизайнерського» тертя**  
   Інструменти (фігури, стрілки, текст, рука, ластик, лазер тощо), контекстне меню, гарячі клавіші та **Command Palette** дають доступ до дій з клавіатури й з панелей без зайвих кроків.

2. **Візуальна мова «sketch»**  
   Рендер через Rough.js підтримує легкий, неформальний вигляд креслень — зручно для пояснень, нотаток і презентацій ідей.

3. **Фокус і мінімалізм інтерфейсу**  
   Режими **Zen** (приховати зайві панелі) та **View** (перегляд без редагування) дозволяють або зосередитись на полотні, або показати схему без ризику випадкових змін.

4. **Спільна робота в реальному часі**  
   Відображення курсорів і змін на полотні, кімнати/посилання для спільної сесії — ключова цінність для командних сценаріїв (реалізовано в `excalidraw-app` поверх редактора).

5. **Портативність і обмін**  
   Експорт у зображення / SVG / JSON, збереження й відкриття сцен, посилання для спільного доступу — щоб артефакт можна було передати або зберегти поза сесією.

6. **Перевикористання контенту**  
   **Бібліотека** елементів (імпорт, збереження наборів) зменшує повторювану роботу при типових фігурах і блоках.

7. **Доступність і локалізація**  
   Підтримка теми (світла/темна/системна), зміна мови інтерфейсу, адаптивність (зокрема мобільна панель інструментів) — щоб редактором можна було користуватись у різних контекстах і на різних пристроях.

8. **Гнучкість для інтеграторів (SDK)**  
   Пропси на кшталт `UIOptions`, `viewModeEnabled`, `zenModeEnabled` і імперативний API дозволяють хост-додаткам підлаштувати видимі дії й режими під свій продукт.

## Які є сценарії

### Робота з полотном

- Створити й редагувати елементи (фігури, лінії, стрілки, текст, зображення, фрейми тощо).
- Виділяти, групувати, вирівнювати, копіювати/вставляти, скасовувати й повторювати дії (**undo/redo**).
- Навігація: масштаб, прокрутка, інструмент «рука», прив’язки та допоміжні лінії під час малювання.
- Пошук елементів на сцені (**Search**), контекстна довідка та підказки на екрані привітання (**Welcome screen**).

### Файли та експорт

- **Завантажити сцену** з файлу / blob; **зберегти** в активний файл (де підтримується).
- **Експорт**: зображення, SVG, JSON; у контексті застосунку — інтеграції на кшталт експорту в хмарні/продуктові напрямки (наприклад Excalidraw+), якщо увімкнено.

### Колаборація та посилання

- Почати **живу колаборацію** (кімната, синхронізація сцени), переглядати стан інших учасників.
- Працювати за **посиланням спільного доступу** (імпорт/синхронізація через бекенд-сервіси клієнта — див. `projectbrief` щодо socket + Firebase).

### Бібліотека та шаблони

- Відкрити **бібліотеку**, додати елементи до неї, імпортувати набори (у т.ч. з URL-токенів, де це реалізовано в додатку).

### Налаштування досвіду

- **Меню** (гамбургер): сцена, експорт, колаборація, палітра команд, пошук, довідка, очищення полотна, соцмережі, преференції, тема, мова, колір фону полотна.
- **Переваги / преференції** редактора (поведінка й відображення згідно з налаштуваннями пакета).

### Вбудовування (інший продукт як хост)

- Розробник вбудовує `<Excalidraw />`, отримує **imperative API**, підписується на `onChange`, обмежує дії через `UIOptions`, вмикає view/zen з пропсів — сценарій «whiteboard у своєму застосунку», а не лише на `excalidraw.com`-подібному сайті.

### Допоміжні сценарії

- Обробка помилок і підтверджень (наприклад перезапис сцени), офлайн/статуси синхронізації файлів у застосунку.
- Для розробки: додаткові пункти меню (наприклад **Visual Debug** у dev-оточенні).
