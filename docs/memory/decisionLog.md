## 2026-03 — Розділення Memory Bank і технічної спеки

- **Контекст**: воркшоп вимагає короткі файли Memory Bank; глибока архітектура не вміщується в ~200 рядків без втрати якості.
- **Рішення**: тримати огляд у `docs/memory/systemPatterns.md`, повну архітектуру — у `docs/technical/architecture.md`; продукт — `docs/product/*`.
- **Наслідки**: два рівні документації; при зміні архітектури спочатку `architecture.md`, потім за потреби оновлювати Memory Bank.

## 2026-03 — Повний набір Memory Bank поза мінімумом грейдингу

- **Контекст**: мінімум для перевірки — три файли в `docs/memory/`; для роботи з проєктом зручні `productContext`, `activeContext`, `progress`, `decisionLog`.
- **Рішення**: додати ці файли як «живу» пам’ять команди; CodeRabbit як і раніше оцінює обов’язковий мінімум окремо.
- **Наслідки**: потрібно дисципліновано оновлювати `activeContext.md` і цей лог.
