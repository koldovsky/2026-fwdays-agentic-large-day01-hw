# Project brief

## Суть продукту

- **Excalidraw** — відкритий whiteboard-інструмент для швидких діаграм, схем і нотаток у **hand-drawn** стилі; орієнтований на мінімальне тертя: швидкий старт, експорт, спільні сесії (у повній збірці застосунку).
- Репозиторій — **npm-монорепозиторій** з пакетами `@excalidraw/*`; **кореневе ім’я workspace** у `package.json`: `excalidraw-monorepo`.
- Два основні способи використання: повноцінний веб-застосунок (`excalidraw-app`) і **вбудовування редактора як React-компонента** через `@excalidraw/excalidraw` (опис у `packages/excalidraw/package.json`: *«Excalidraw as a React component»*).

## Цінність для користувача

- **Швидкість і простота:** накреслити схему під обговорення без довгого онбордингу; характерний неформальний вигляд ліній і тексту знижує бар’єр «ідеального» документа.
- **Контроль даних:** сцени зберігаються як JSON, експорт у PNG/SVG; у застосунку можливі колаборація та обмін посиланнями (залежно від конфігурації хоста).
- **Інтеграція:** React-розробники вбудовують редактор у свої продукти через публічний API (`useExcalidrawAPI`, меню, колбеки), не форкуючи весь застосунок.

## Основна мета

- Надати **інтерактивний редактор креслень** (елементи на canvas, експорт, бібліотека, колаборація тощо).
- Розділити код на шари:
  - **`packages/common`** — спільні константи й утиліти.
  - **`packages/element`** — модель і операції над елементами.
  - **`packages/math`** — геометрія для canvas.
  - **`packages/utils`** — додаткові утиліти.
  - **`packages/excalidraw`** — UI редактора, дані, i18n, публічний API.
  - **`excalidraw-app`** — оболонка продукту (деплой, колаборація, інтеграції); точка входу `excalidraw-app/index.tsx` рендерить `ExcalidrawApp` з `./App`.

## Межі scope (бібліотека vs застосунок)

- **`@excalidraw/excalidraw`** постає **бібліотеку редактора**: компонент, типи, i18n, експорт/імпорт сцени з клієнта. Інтегратори підключають залежності через npm і керують хостингом самі.
- **`excalidraw-app`** додає **продуктовий шар**: збірка з Vite, колаборація (Socket.IO, Firebase-клієнти в репозиторії), Sentry, PWA-облога — те, що потрібно для розгорнутого сайту, а не для мінімального embed.

## Non-goals (рівень продукту, коротко)

- Не **CAD** і не система креслення з точними допусками; hand-drawn естетика — свідома властивість.
- Не **повна заміна** Figma/Sketch для pixel-perfect UI або дизайн-систем; фокус — швидкість і діаграми, а не дизайн-макети production-рівня.
- **Серверний бекенд** для AI-фіч не входить у цей OSS-репозиторій; опційні зовнішні інтеграції описані в продуктовій документації, а не як обов’язкова частина пакета.

## Приклади для інтеграторів
